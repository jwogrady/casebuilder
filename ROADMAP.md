# Roadmap

## M0 — Product foundation

Goal: make the product definition explicit.

- Product brief
- Architecture brief
- Artifact contracts
- ADR structure
- Repository scaffold
- Package boundaries

## M1 — Build the case

Goal: produce a verified Canonical Case Record from a Capture Bundle.

- Capture Bundle ingestion
- PDF explosion
- Page rendering
- Text extraction
- Page transcription
- Provenance chain
- Quality metadata
- Filesystem projection
- DuckDB projection
- Verification gate

## M2 — Understand the case

Goal: turn the verified record into human-reviewable understanding.

- Timeline reconstruction
- Financial reconstruction
- Issue map
- Evidence map
- Missing evidence detection
- Contradiction detection
- Exhibit organization
- Authority organization
- Human review packets

## M3 — Generate the case

Goal: create work product only from approved understanding.

- Motion drafting
- Response drafting
- Declarations
- Discovery requests
- Exhibit packets
- Appendices
- E-filing packets
- Filing-to-capture loop

## M4 — Operating system for litigation

Goal: make every matter continuously improve.

- Matter workspace
- Versioned record history
- Change detection
- Review deltas
- Filing strategy history
- Multi-case patterns
- Team collaboration
